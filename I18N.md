# Internationalization (i18n) Documentation

OpenMES supports multi-language functionality using Laravel's built-in translation system.

## Supported Languages
- **English (en)**: Default and fallback language.
- **Turkish (tr)**: Fully supported secondary language.

## Architecture
The system uses **JSON-based translation files** located in `backend/lang/`.
- `backend/lang/en.json`
- `backend/lang/tr.json`

A global middleware `SetLocaleMiddleware` detects the requested language from the `Accept-Language` HTTP header and sets the application locale accordingly.

## Adding a New Language
To add a new language (e.g., German - `de`):

1.  **Create JSON file**: Create `backend/lang/de.json`.
2.  **Copy Keys**: Copy all keys from `en.json` to `de.json`.
3.  **Translate Values**: Replace English values with German translations.
4.  **Update Middleware**: Add `'de'` to the allowed languages array in `app/Http/Middleware/SetLocaleMiddleware.php`:
    ```php
    if (in_array($lang, ['en', 'tr', 'de'])) {
        App::setLocale($lang);
    }
    ```

## Usage in Code

### Backend (PHP)
Always use the `__()` helper for user-facing strings:
```php
return response()->json([
    'message' => __('Work order created successfully')
]);
```

### Response Format
Localized responses include a `locale` field at the root level (implemented via `StandardApiResponse` trait) to confirm the active language:
```json
{
    "status": "success",
    "message": "İş emri başarıyla oluşturuldu",
    "locale": "tr",
    "data": { ... }
}
```

## Best Practices
1.  **Do NOT hardcode strings**: Never use raw strings in controllers, services, or views.
2.  **Identical Keys**: Ensure all JSON translation files contain exactly the same keys.
3.  **Fallback**: If a key is missing in a specific language file, Laravel will automatically fallback to the value in the English file (or the key itself).
